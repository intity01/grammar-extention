## üåê Chrome Web Store Submission

### Step 1: Create Developer Account

1. **Go to Chrome Web Store Developer Dashboard:**
   https://chrome.google.com/webstore/devconsole

2. **Sign in with Google account**

3. **Pay one-time $5 developer registration fee**
   - Required for first-time developers
   - One-time payment, valid forever
   - Payment via credit card

4. **Accept Developer Agreement**

### Step 2: Create New Item

1. **Click "New Item" button**

2. **Upload ZIP file:**
   - Select `grammar-checker-extension.zip`
   - Wait for upload to complete
   - System will validate the package

3. **If validation fails:**
   - Check error messages
   - Fix issues in code
   - Rebuild and repackage
   - Upload again

### Step 4: Privacy Practices

**Privacy Policy URL (Required):**

```
https://intity01.github.io/grammar-extention/
```

**Single Purpose Description:**

```
This extension provides real-time grammar checking for Thai, English, and Japanese languages using local processing.
```

**Permission Justifications:**

**storage:**

```
Store user settings and language dictionaries locally for offline grammar checking.
```

**activeTab:**

```
Access text content in the active tab to perform grammar checking when the user activates the extension.
```

**offscreen:**

```
Create an offscreen document for clipboard operations in the clipboard correction mode.
```

**clipboardWrite:**

```
Allow users to copy corrected text to clipboard when using clipboard correction mode.
```

**contextMenus:**

```
Add a "Check Grammar" option to the right-click context menu for convenient access.
```

**notifications:**

```
Display notifications to inform users about grammar check results and extension status.
```

**host_permissions (<all_urls>):**

```
Enable grammar checking on all websites including Gmail, Google Docs, social media, and any text input fields.
```

**Data Usage:**

- Select: **This extension does not collect user data**

### Step 6: Submit for Review

1. **Review all information**
   - Double-check all fields
   - Verify screenshots are correct
   - Ensure privacy policy URL works

2. **Click "Submit for Review"**

3. **Wait for review:**
   - Typical review time: 1-3 business days

4. **If rejected:**
   - Read rejection reason carefully
   - Fix the issues
   - Resubmit

## üìä Review Process

### Chrome Web Store

**Timeline:**

- Initial review: 1-3 business days
- Updates: Usually faster (1-2 days)

**Common Rejection Reasons:**

1. Privacy policy issues
2. Permission justifications unclear
3. Functionality doesn't match description
4. Code obfuscation or minification issues
5. Malware or policy violations

**Status Tracking:**

- Dashboard: https://chrome.google.com/webstore/devconsole
- Email notifications
- Status: Pending ‚Üí In Review ‚Üí Published/Rejected

### Microsoft Edge Add-ons

**Timeline:**

- Initial review: 3-7 business days
- Updates: 2-5 business days

**Common Rejection Reasons:**

1. Privacy policy missing or inadequate
2. Permissions not justified
3. Functionality issues
4. Store listing incomplete
5. Policy violations

**Status Tracking:**

- Partner Center: https://partner.microsoft.com/dashboard/microsoftedge
- Email notifications
- Status: In Review ‚Üí Certification ‚Üí Published/Failed

## üìù Store Listing Best Practices

### Name

- Keep it clear and descriptive
- Include key features (languages)
- Stay within 45 characters

### Description

- Start with key benefits
- Use bullet points for features
- Include screenshots references
- Mention privacy/offline features
- Add call-to-action

### Screenshots

- Show actual functionality
- Use high resolution (1280x800)
- Add captions if helpful
- Show all three languages
- Highlight key features

### Keywords

- Use relevant search terms
- Include language names
- Add feature keywords
- Use native language terms
- Don't keyword stuff

## üÜò Troubleshooting

### Submission Rejected

**Privacy Policy Issues:**

- Ensure URL is accessible
- Check policy covers all permissions
- Verify HTTPS connection

**Permission Issues:**

- Justify each permission clearly
- Explain why it's necessary
- Link to privacy policy

**Functionality Issues:**

- Test extension thoroughly
- Ensure description matches functionality
- Fix any bugs before resubmitting

### Review Taking Too Long

**Chrome (>5 days):**

- Check dashboard for messages
- Contact Chrome Web Store support
- Be patient, reviews can be delayed

**Edge (>10 days):**

- Check Partner Center for updates
- Contact Edge Add-ons support
- Reviews can take longer
